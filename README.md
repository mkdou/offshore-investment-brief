# 境外投资每日内参看板

一个可部署到 GitHub Pages 的静态网页，用于沉淀港美股、海外市场、打新生态、监管合规和跨境投资平台生态动态。

它不是固定日报汇报模板，而是个人工作用的境外投资资讯资料库：日常可检索，临时需要沟通时能快速找到来源、事实和研判。

## 本地预览

```bash
python3 -m http.server 4173
```

然后打开：

```text
http://localhost:4173
```

## 日常更新

主要内容在 `data/daily.json`。不要覆盖旧日报，建议每天往 `reports` 数组最前面追加一条：

- `topSignals`：当天最重要的 2-4 个信号
- `marketPulse`：全球市场与港美股环境
- `ipoPipeline`：港美股打新市场公开动态，需要包含交易所、公告或招股书等来源
- `ecosystemMoves`：互联网大厂、跨境平台、同业券商动态
- `regulatoryWatch`：CSRC / SFC / SEC 等监管合规信息
- `actions`：资料库、专题研究或后续复盘动作
- `sources`：本期公共来源，优先官网、交易所、监管机构、公司公告

网页支持日期切换、跨日报模糊搜索、来源链接跳转。临时研判区只保存在浏览器本地；点击“导出当前日报”会导出当前日期的数据，方便你改完后再合并回 `data/daily.json`。

## 数据原则

- 不接入公司内部接口。
- 不记录我司 App 额度、预约、内部运营数据。
- 打新只记录公开市场数据，例如交易所公告、招股书、认购倍数、暗盘、首日表现、行业分布、同业公开动作。
- 研判尽量写成“为什么重要 / 后续观察什么”，避免泛泛而谈的产品建议。
- 重要资讯按“事实 -> 证据强度 -> 触发阈值 -> 下一步验证”写。强证据优先使用交易所文件、监管公告、公司公告、财报、招股书；媒体和社交讨论只作为线索。
- 平台生态类资讯按“系统变化 -> 稀缺环节 -> 竞争影响 -> 反证条件”拆，不只记录新闻标题。

## GitHub Pages 部署

1. 将本目录推送到 GitHub 仓库。
2. 进入仓库 Settings -> Pages。
3. Source 选择 `Deploy from a branch`。
4. Branch 选择 `main`，目录选择 `/root`。
5. 保存后等待 Pages 生成访问链接。

这是纯静态页面，不接入任何公司内部数据接口。
